01

Terminal Operations

Control and monitor Claude Code terminals directly from Claude Desktop. Read output, send commands, search logs.

5 TOOLS
bridge_read_terminal
Read recent output from a Claude Code terminal. Essential for seeing what Claude Code is doing without switching apps.
Parameters
project_id string Project to read from (default: active)
lines number Lines to read, 1-500 (default: 50)
"Show me the last 100 lines from my terminal"
bridge_send_text
Send text input to a Claude Code terminal. Type text as if at the keyboard. Can optionally press Enter after.
Parameters
text string Text to send (max 1000 chars)
press_enter boolean Press Enter after (default: true)
"Run 'npm test' in my terminal"
bridge_send_key
Send special keys like Enter, Ctrl+C, Escape, Tab, and arrow keys to the terminal.
Parameters
key enum enter, ctrl-c, ctrl-d, ctrl-z, escape, tab, up, down
"Press Ctrl+C to stop the process"
bridge_search_output
Search terminal output for patterns using regex. Find errors, warnings, or any text in recent output.
Parameters
pattern string Regex pattern (e.g., 'error|warning')
lines number Lines to search (default: 100)
"Search for errors in my terminal output"
bridge_sync_remote
Sync logs from remote SSH projects. Fetch latest terminal output from the server.
Parameters
project_id string Remote project to sync
"Sync my remote server logs"
02

Project Management

Manage multiple Claude Code projects across local and remote environments. Switch contexts, monitor status, add new projects.

6 TOOLS
bridge_list_projects
List all configured projects. Shows type (local/remote), path, and status. Indicates which is active.
Parameters
response_format enum markdown or json
"Show me all my projects"
bridge_add_project
Add a new Claude Code project. Configure for local Mac terminal or remote SSH connection.
Parameters
id string Unique ID (lowercase, hyphens)
name string Human-readable name
type enum local or remote
path string Project directory path
"Add my project at ~/Code/my-app"
bridge_remove_project
Remove a project from the bridge. Removes configuration only — does not delete actual files.
Parameters
project_id string Project to remove
"Remove the old-project config"
bridge_switch_project
Switch to a different active project. Changes which project receives commands by default.
Parameters
project_id string Project to activate
"Switch to my backend project"
bridge_get_project_status
Get detailed status including running state, recent activity, error counts, and pending approvals.
Parameters
project_id string Project to check (default: active)
"What's the status of my project?"
03

Approval System

Review and authorize risky operations before execution. Complete control over what Claude Code can run.

4 TOOLS
bridge_list_pending
List all pending approval requests. Shows commands waiting for your review with context and risk levels.
Parameters
project_id string Filter by project (optional)
"What's waiting for my approval?"
bridge_request_approval
Submit a command for approval. Typically called by Claude Code when it wants to run something risky.
Parameters
command string Command requesting approval
type enum command, file_write, file_delete, network, install
context string Why this command is needed
(Called programmatically by Claude Code)
bridge_approve
Approve a pending request. Marks as approved and optionally sends 'y' to the terminal to proceed.
Parameters
request_id string Request to approve
execute boolean Also send 'y' to terminal (default: true)
"Approve that database migration"
bridge_deny
Deny a pending request. Marks as denied and optionally sends 'n' to the terminal.
Parameters
request_id string Request to deny
reason string Optional denial reason
"Deny that request, too risky"
04

Auto-Approve Rules

Automatically approve trusted command patterns. Build rules for common operations to reduce interruptions.

4 TOOLS
bridge_list_rules
List all auto-approve rules. Shows patterns that automatically approve matching commands.
Parameters
response_format enum markdown or json
"Show my auto-approve rules"
bridge_add_rule
Add an auto-approve rule. Commands matching the pattern are automatically approved (except critical risk).
Parameters
name string Human-readable name
pattern string Regex pattern to match
path_whitelist string[] Only approve if path matches
expires_hours number Hours until rule expires
"Auto-approve all npm commands"
bridge_remove_rule
Remove an auto-approve rule. The rule will no longer match commands.
Parameters
rule_id string Rule to remove
"Remove the npm auto-approve rule"
bridge_toggle_rule
Enable or disable an auto-approve rule without removing it.
Parameters
rule_id string Rule to toggle
enabled boolean Enable or disable
"Disable the git rule temporarily"
05

Smart Alerts

Get notified when specific patterns appear in terminal output. Never miss another error or warning.

4 TOOLS
bridge_watch_for
Set up a pattern alert. Watch for specific regex patterns in terminal output and get notified.
Parameters
pattern string Regex pattern (e.g., 'error|failed')
alert_id string Custom ID (auto-generated if omitted)
"Alert me if you see any errors"
bridge_stop_watching
Remove a pattern alert. The pattern will no longer be monitored.
Parameters
alert_id string Alert to stop
"Stop watching for errors"
bridge_get_alerts
Get all alerts or only triggered alerts. See which patterns have matched.
Parameters
triggered_only boolean Only show triggered alerts (default: false)
"Show me any triggered alerts"
bridge_clear_alerts
Reset triggered alerts so they can trigger again. Does not remove the alerts.
Parameters
project_id string Clear for project (or all)
"Clear all triggered alerts"
06

Behavior Watchdog

Detect and stop AI shortcuts, laziness, and placeholder code. 20+ patterns monitor for common AI mistakes.

6 TOOLS
bridge_watchdog_status
Get watchdog status. Shows enabled state, pattern count, violation count, and all detection patterns.
Returns
enabled boolean Whether watchdog is active
pattern_count number Total patterns (default + custom)
violation_count number Total violations detected
"What's the watchdog status?"
bridge_watchdog_enable
Enable the watchdog. Starts monitoring all terminals every 2 seconds. Sends ESC on violations.
When Enabled
Polls terminals every 2 seconds
Checks against 20+ patterns
Sends ESC to stop Claude on violations
"Enable the behavior watchdog"
bridge_watchdog_disable
Disable the watchdog. Stops monitoring. Existing violations remain in history.
Parameters
No parameters required
"Turn off the watchdog"
bridge_watchdog_add_pattern
Add a custom detection pattern. Define additional behaviors to monitor beyond defaults.
Parameters
name string Pattern name
pattern string Regex pattern (case-insensitive)
description string What this pattern detects
"Add a pattern for 'I'll create a stub'"
bridge_watchdog_remove_pattern
Remove a custom pattern. Only custom patterns can be removed — defaults are protected.
Parameters
pattern_id string Pattern to remove
"Remove the stub pattern"
bridge_watchdog_violations
Get violation history. See what shortcuts were caught, when, and what text triggered them.
Parameters
limit number Max violations to return
"Show me all watchdog violations"

Task Bible

Structured task management with persistence across sessions. Claude always knows what's done and what's next.

5 TOOLS
bridge_get_task_bible
Get the active task bible. Returns current task list with progress for a project.
Parameters
project_id string Project to get tasks for
bible_id string Specific task bible ID
"Show me my task list"
bridge_get_next_task
Get the next task to work on. Respects dependencies and priorities to suggest what's next.
Parameters
bible_id string Task Bible ID
"What should I work on next?"
bridge_update_task_status
Update a task's status. Mark as in_progress, completed, blocked, or cancelled.
Parameters
task_id string Task to update
status enum pending, in_progress, completed, blocked, cancelled
notes string Optional notes about status change
"Mark task 3 as complete"
bridge_add_task_note
Add a progress note to a task. Document work done or issues encountered.
Parameters
task_id string Task ID
note string Note to add
"Add a note: 'Found edge case in auth'"
bridge_check_dependencies
Check task dependencies. See what must be completed before a specific task can start.
Parameters
task_id string Task to check dependencies for
"What blocks task 5?"
08

Setup Wizard

Guide users through 0711 Medusa configuration. Auto-detect projects, test API keys, verify installation.

5 TOOLS
setup_detect_folders
Automatically detect project folders. Scans Desktop, Documents, Code for development folders.
Parameters
includeHidden boolean Include hidden folders (default: false)
recursive boolean Scan subdirectories (default: false)
maxDepth number Max recursive depth (default: 2)
"Find my project folders"
setup_add_folder
Add a project folder to Medusa configuration. Configure with custom name and terminal app.
Parameters
folderPath string Absolute path to folder
projectName string Custom name (optional)
terminalApp enum iTerm2 or Terminal.app
"Add ~/Code/myapp to Medusa"
setup_get_status
Check overall setup status. See what's configured and what steps remain.
Returns
Claude Desktop installed/configured
Projects configured count
API keys status
Completion percentage
"How's my setup looking?"
setup_run_complete
Run automated setup process. Checks Claude Desktop, configures MCP server, restarts if needed.
Parameters
autoRestart boolean Restart Claude Desktop automatically
"Run the full setup"
setup_test_api_key
Validate an API key before saving. Test OpenAI, Google AI, or Anthropic keys.
Parameters
provider enum openai, google, or anthropic
apiKey string Key to test
"Test my OpenAI API key"
